I recently came across this Monte Carlo Stick Problem problem which was neatly summarised for me. Luckily I had spent a little bit of time looking into Monte Carlo simulations prior to this so I had an idea of what was expected. To summarise the problem

Let there be a stick of length 1. Pick two points uniformly at random along the stick, and break the stick at those points. What is the probability of the three resulting pieces being able to form a triangle?

At first glance it all seemed very daunting. My first thought was how do I know if three lines form a triangle. I could have looked that up on the internet, as I could have done with this whole problem, but where is the fun in that. I remember that I ran into this problem a while back, but cannot remember where. I will link to it when I remember. I was thinking about whether the lines could cross, what the angels would have to be, etc. It was during my lunch-time swim that I realised a simple way to determine whether three lines could form a triangle.

Determining if three lines form a triangle

  • Firstly, let us talk about the three line lengths as A, B, C.
  • Let us put a constraint on the lengths such that A > B > C.
  • Now A is always the longest line, C the smallest.
  • If B + C < A then together B + C will not be able to reach both ends of point of A.
  • If B + C cannot reach both ends of A, then it definitely will not be able to make a triangle.
  • So A, B, C will make a triangle if A < B + C.

I ended up removing the constraint A > B > C and calculated it by using a priority queue. Below is the method I used to determine if three lengths form a triangle.

static private Boolean isTriangle(Double side1, Double side2, Double side3) {

    PriorityQueue sides = new PriorityQueue(3);

    sides.add(side1);

    sides.add(side2);

    sides.add(side3);



    return sides.remove() + sides.remove() > sides.remove();

}

Optimization

The constraint on this problem is that the line is of length 1. This means that the code shown above can be simplified. Knowing that there is an upper bound for the length of the line and that it is 1, in this case, you could just check to see that the longest line segment is not larger than 0.5. More generically

return Math.max(side1, side2, side3) < (side1 + side2 + side3) * 0.5;

Splitting the line

The next part of the problem was splitting the line into segments. Luckily, the first case was simply choosing two points uniformly in the range [0 ; 1]. I used Math.random() to get the uniformly random numbers. There is a slight issue with this, Math.random() in java gives a value in the range [0 ; 1).  But what does it mean to select 0 as a split point or 1 for that matter. Selecting 0 will not split the line as there is nothing smaller than 0 and no split will occur, you will effectively end up with two lines, or technically three with one line of length 0. So, the way I have implemented it has an unhandled edge case which is when Math.random() gives me 0. Putting that aside for the moment, how does one split the line?

If you select two points x, y in the range (0 ; 1) and you split a line of length 1 at those intercepts, then

  • Assuming x < y
  • Length A = length of 0 to x, which = x
  • Length B = length of x to y, which = y - x
  • Length C = length of y to 1, which = 1 - y

I removed the constraint x < y when implementing the solution. The below code is how I split the line at two points, split1 and split2.

isTriangle(Math.min(split1, split2), Math.abs(split1 - split2), 1 - Math.max(split1, split2))

The probability of a triangle (Monte Carlo)

I know there is a way to analytically come to an answer, however my math fails me when trying. To do this by way of simulation is rather simple.

  1. All you need to do is randomly choose two points
  2. Split the line at those two points
  3. Test whether it is a triangle
  4. Repeat steps 1 through 3 a large amount of times (X), keeping track of all triangles found (Y)
  5. Then the probability of there being a triangle when breaking a line at two uniformly random points is = Y / X.

I did something similar to this a while back when calculating poker hand strength. Sadly, it was before this blog and I never posted the code. Essentially, there I would play X amount of games with the current hand and visible cards to see how many times the hand won (Y). The hand strength was the probability of winning which was Y / X.

Modifying the distribution

For the third question I had to modify the distribution when selecting the random split points. I have fiddled with distributions in the past, however that was with dice mostly. The way I achieved it here was

  1. Determine probability intercepts satisfying the distribution requirement. In this case 0.25, 0.75. This means that the first range is [0 ; 0.25), second [0.25 ; 0.75), and third [0.75, 1)
  2. Select a number Y uniformly distributed between [0 ; 1)
  3. Determine which range it falls into.
  4. In this case it will fall 25% of the time into [0 ; 0.25)
  5. 50% of the time into [0.25 ; 0.75)
  6. 25% of the time into [0.75 ; 1)
  7. Then generate a number uniformly in the range determined above.

I implemented this with a simple if, else tree. However, I am sure there must be a neater way to handle this. If I spent more time on this I am certain I would find a nicer way to handle it. Below is the code I wrote to pull numbers from a modified distribution.

public Double select() {

    Double probability = Math.random();

    if (probability < .25) { // 25% first third

        return Math.random() / 3;

    } else if (probability < .5) { // 25% last third

        return 1 - Math.random() / 3;

    } else { // 50% middle third

        return Math.random() / 3 + 1 / 3;

    }

}

The answers

Splitting a line at two uniformly distributed points in the range [0 ; 1) will result in being able to make a triangle 25% of the time.

Splitting a line at two points that suit the modified distribution will result in being able to make a triangle 4.6% of the time. My assumption is that selecting split points close to each end of the original line would result in a large middle section which is at most times larger than half of the original line length.

The code

I ended up refactoring a bit whilst coding. I wish I could have used functional programming, but Java has it's limitations. The lambda expression in JDK8 will definitely make this code neater. I have created a gist of this code as well.

package interview.code;



import java.util.PriorityQueue;



public class MonteCarloStick {



    static public void main(String[] args) {

        System.out.println("uniform split, triangle probability \t= " + triangleProbability(100000, new ILineSplit() {

            @Override

            public Double select() {

                return Math.random();

            }

        }));



        System.out.println("modified split, triangle probability \t= " + triangleProbability(100000, new ILineSplit() {

            @Override

            public Double select() {

                Double probability = Math.random();

                if (probability < .25) { // 25% first third

                    return Math.random() / 3;

                } else if (probability < .5) { // 25% last third

                    return 1 - Math.random() / 3;

                } else { // 50% middle third

                    return Math.random() / 3 + 1 / 3;

                }

            }

        }));

    }



    static private Double triangleProbability(Integer iterations, ILineSplit lineSplit) {

        Integer numTriangles = 0;

        Double split1;

        Double split2;



        for (int count = 0; count < iterations; count++) {

            split1 = lineSplit.select();

            split2 = lineSplit.select();

            if (isTriangle(Math.min(split1, split2), Math.abs(split1 - split2), 1 - Math.max(split1, split2))) {

                numTriangles++;

            }

        }



        return numTriangles.doubleValue() / iterations.doubleValue();

    }



    static private Boolean isTriangle(Double side1, Double side2, Double side3) {

        PriorityQueue sides = new PriorityQueue(3);

        sides.add(side1);

        sides.add(side2);

        sides.add(side3);



        return sides.remove() + sides.remove() > sides.remove();

    }

}



interface ILineSplit {



    Double select();

}
